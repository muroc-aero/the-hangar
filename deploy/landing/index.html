Servers

Three MCP servers, each wrapping a proven open-source analysis framework.

OAS: OpenAeroStruct

--

Aerostructural analysis and optimization of lifting surfaces. Couples vortex-lattice aerodynamics with finite-element structures for wing design.

~20 tools · geometry, aero analysis, aerostructural analysis, drag polars, stability derivatives, optimization
https://mcp.lakesideai.dev/oas/mcp

OCP: OpenConcept

--

Aircraft conceptual design and mission analysis. Computes fuel burn, takeoff performance, and battery state-of-charge for conventional and hybrid-electric architectures.

~15 tools · aircraft templates, propulsion architecture, mission analysis, parameter sweeps, optimization
https://mcp.lakesideai.dev/ocp/mcp

PYC: pyCycle

--

Gas turbine engine cycle analysis. Design-point sizing and off-design performance evaluation with full thermodynamic station modeling.

~13 tools · engine creation, design-point analysis, off-design sweeps, TSFC & thrust computation
https://mcp.lakesideai.dev/pyc/mcp

Connect

Add the servers to your AI client in seconds.

claude mcp add --transport http oas https://mcp.lakesideai.dev/oas/mcp
claude mcp add --transport http ocp https://mcp.lakesideai.dev/ocp/mcp
claude mcp add --transport http pyc https://mcp.lakesideai.dev/pyc/mcp

Start a Claude Code session. The tools are available immediately.

1. Go to claude.ai

2. Settings → Integrations → Add MCP Server

3. Enter the URL for the tool you want:

https://mcp.lakesideai.dev/oas/mcp
https://mcp.lakesideai.dev/ocp/mcp
https://mcp.lakesideai.dev/pyc/mcp

4. Sign in when redirected to Keycloak. Repeat for each tool.

codex mcp add oas --url https://mcp.lakesideai.dev/oas/mcp
codex mcp add ocp --url https://mcp.lakesideai.dev/ocp/mcp
codex mcp add pyc --url https://mcp.lakesideai.dev/pyc/mcp

# Authenticate each server:
codex mcp login oas --scopes mcp:tools
codex mcp login ocp --scopes mcp:tools
codex mcp login pyc --scopes mcp:tools
All servers use Keycloak OIDC. On first connection, your browser will open for login. You can sign in with Google or credentials provided by an admin.

What Is The Hangar?

March 2026 · Lakeside AI

The Hangar is an open-source collection of MCP servers that wrap real aerospace analysis solvers. AI agents can call these tools directly to compute aerodynamic coefficients, structural loads, fuel burn, engine performance, and more. No approximations: the agent runs the same solvers you would run yourself.

Background

Tools like OpenAeroStruct, OpenConcept, and pyCycle are capable but have significant setup overhead. Building an OpenMDAO problem, defining meshes, connecting subsystems, setting solver options, and extracting results takes time and domain knowledge. This makes quick design iteration expensive, especially early in a project when you need to explore broadly.

MCP (Model Context Protocol) is an open standard that lets AI applications call external tools with structured inputs and outputs. The Hangar uses MCP to expose each analysis framework as a set of callable tools with input validation, physics checks, and structured result envelopes.

How It Works

Each server exposes tools that map to common engineering workflows:

  • Define geometry: create a wing surface with span, chord, sweep, taper, and thickness distributions
  • Run analysis: compute CL, CD, structural stresses, fuel burn, TSFC, or thrust at specified flight conditions
  • Optimize: gradient-based optimization over design variables with engineering constraints
  • Sweep parameters: drag polars, off-design engine maps, mission range trades

Every tool call returns a structured envelope containing results, physics validation findings (convergence, bounds, sanity checks), and a run ID for provenance tracking.

The Tool Chain

All three servers build on the OpenMDAO ecosystem from NASA Glenn:

  • OpenAeroStruct (OAS): vortex-lattice method coupled with finite-element structures. Suited for wing planform studies, aerostructural trade-offs, and configuration screening.
  • OpenConcept (OCP): mission analysis with full flight profiles. Supports turboprop, turbofan, and series-hybrid-electric architectures for fuel burn and range evaluation.
  • pyCycle (PYC): thermodynamic cycle analysis for gas turbines. Design-point engine sizing and off-design performance with component-level modeling.

Example Workflows

Some things you can do with an AI agent connected to these servers:

  • Design space exploration: "What happens to fuel burn if aspect ratio goes from 9 to 14?" The agent creates the surface, runs analyses at each AR, and reports the trend.
  • Architecture comparison: "Compare turboprop vs. series-hybrid for a 500 nmi regional mission." Computed results across the full mission profile, not estimates.
  • Multi-tool analysis: size an engine with pyCycle, feed its TSFC into an OpenConcept mission model, and optimize the wing with OpenAeroStruct.
  • Optimization: define design variables, set constraints (stress, fuel capacity, CL targets), run SNOPT or SLSQP, and interpret convergence.

The agent handles the setup boilerplate. You stay focused on the engineering questions.

Contribute

The Hangar is designed to support new analysis tools beyond the initial three.

We welcome contributions of MCP server wrappers for other engineering tools. The SDK provides shared infrastructure for response envelopes, provenance tracking, validation, and artifact storage, so you can focus on the tool-specific logic.

Check the issues and discussions on GitHub to see what tools are being considered, propose new ones, or pick up an existing request.
